Project Overview

Deep dive into what FoodVision AI is and how it works under the hood.

What is this project?

FoodVision AI bridges the gap between state-of-the-art computer vision and everyday dietary tracking. By simply uploading an image of a meal, the system instantly identifies the food type and retrieves its complete macronutrient and micronutrient profile.

Built with scalability and rapid inference in mind, the backend leverages Redis caching to ensure O(1) response times for identical or repeated image queries, bypassing the expensive neural network forward pass. The frontend features a modern, responsive Glassmorphism UI for a premium user experience.

What does it do?

The system accepts a food image and classifies it into one of 34 food categories. Along with the prediction, it provides:

  • Predicted food class with confidence score
  • Top 5 prediction probabilities ranked
  • Full nutritional breakdown (calories, protein, carbs, fat, fiber, sugar)
  • Vitamins (A, C) and minerals (calcium, iron, potassium)
  • Model performance metrics (accuracy, precision, recall, F1)
  • Confusion matrix stats (TP, TN, FP, FN)
  • Redis-cached results for instant repeated lookups

Deep Learning Architectures

Three distinct CNNs trained to demonstrate the power of Transfer Learning.

Baseline
Custom CNN

A sequential architecture built from scratch with interleaved Conv2D (ReLU) and MaxPooling2D layers, flattened into fully connected Dense networks with heavy Dropout to combat overfitting.

Accuracy ~24%
Establishes dataset difficulty and the necessity for deeper architectures.
Transfer Learning
VGG16

Utilizes the deep VGG16 convolutional base frozen on ImageNet. A custom classification head with GlobalAveragePooling2D and Dense layers is fine-tuned on 34 food classes.

Accuracy ~51%
Massive leap in spatial feature recognition over baseline.
★ Recommended
ResNet

Employs identity shortcut connections that skip layers, solving the vanishing gradient problem. Uses ImageNet weights with a fine-tuned top dense classification head.

Accuracy Best Overall
Highest precision, recall stability, and robustness across food types.

Development & Architecture

Under the hood — how FoodVision AI was engineered.

Model Training Pipeline

  • Trained using TensorFlow / Keras 2.16 deep learning framework
  • 3 architectures: Custom CNN, VGG16, and ResNet with Transfer Learning
  • Input resolution: 256 × 256 pixels (RGB)
  • Data augmentation via ImageDataGenerator — rotation, flipping, zoom, shifting
  • Comprehensive validation with per-class TP, TN, FP, FN metrics
  • Models saved as .h5 files and auto-downloaded via Google Drive

Application Architecture

  • Built with Object-Oriented Programming principles from the ground up
  • 6 modular utility classes: ModelLoader, ImagePreprocessor, Predictor, MetricsReader, NutritionLoader, RedisCache
  • Flask Application Factory pattern for clean initialization
  • Redis caching layer with SHA256-based image fingerprinting
  • In-memory image processing — zero disk writes for privacy
  • Deployed on Render with Gunicorn production server
